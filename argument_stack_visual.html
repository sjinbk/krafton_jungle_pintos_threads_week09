Pintos 인자 전달 스택 그림

예시는 echo hello world를 실행한다고 가정합니다. 스택은 높은 주소에서 낮은 주소로 자라기 때문에, 아래 그림의 위쪽이 더 높은 주소입니다.

코드 흐름

argc = 3 argv[0] = "echo" argv[1] = "hello" argv[2] = "world"
문자열을 뒤에서부터 스택에 복사합니다. 복사 후 argv_addr[i]에는 각 문자열이 놓인 유저 스택 주소가 저장됩니다.
if_->rsp = (void *)((uint64_t) if_->rsp & ~0x7)로 8바이트 정렬을 맞춥니다.
argv_addr[argc] = NULL까지 포함해서, 문자열 주소들을 다시 스택에 넣습니다. 이 구간이 유저 프로그램의 argv 배열입니다.
start_addr = if_->rsp는 argv 배열의 시작 주소를 기억합니다.
fake return address로 NULL을 하나 더 넣습니다.
rdi = argc, rsi = start_addr로 유저 프로그램에 인자를 넘깁니다.

지금 선택한 코드의 char *argv_addr[...] = NULL;은 C에서 배열 초기화로는 맞지 않습니다. char *argv_addr[...] = { NULL }; 또는 argv_addr[argc] = NULL;을 따로 넣는 형태가 좋습니다.

포인터 해석

여기서 헷갈리는 핵심은 if_->rsp가 타입 있는 변수가 아니라 “스택의 현재 주소값”이라는 점입니다. 그래서 그 주소에 어떤 종류의 값을 쓸지 C에게 캐스팅으로 알려줘야 합니다.

char *

문자 하나의 주소입니다. C 문자열은 첫 글자의 주소로 표현되므로 "hello"를 가리키는 값의 타입이 char *입니다.

char **

char * 값을 저장하는 칸의 주소입니다. 즉 argv[0] 같은 문자열 포인터 하나를 써 넣으려면 그 저장 위치를 char **로 봅니다.

void **

void * 값을 저장하는 칸의 주소입니다. fake return address 자리에는 특정 타입의 포인터가 아니라 그냥 포인터 값 NULL을 넣으므로 이렇게 볼 수 있습니다.

if_->rsp -= sizeof(char *); *(char **) if_->rsp = argv_addr[i];
왼쪽 (char **) if_->rsp: 이 주소를 char * 저장 칸의 주소로 해석
오른쪽 argv_addr[i]: 실제로 저장할 char * 값, 즉 문자열이 있는 주소
결과 if_->rsp 위치의 8바이트 칸에 문자열 주소 하나가 저장됨
if_->rsp -= sizeof(void *); *(void **) if_->rsp = NULL;
왼쪽 (void **) if_->rsp: 이 주소를 void * 저장 칸의 주소로 해석
오른쪽 NULL: 저장할 포인터 값. 메모리에는 보통 0x0000000000000000이 들어감
결과 NULL 포인터의 주소를 넣는 것이 아니라, NULL 포인터 값 자체를 스택 칸에 저장함

*(char **) if_->rsp에서 별표가 많아 보여도 “포인터가 여러 개 저장된다”는 뜻은 아닙니다. if_->rsp 주소에 있는 8바이트 칸 하나를 골라서, 그 칸에 포인터 값 하나를 쓰는 표현입니다.
